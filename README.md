# CTS Website

This repository contains the public website for **CTS — Cognition Topology Stack**.

The site explains CTS, presents selected architectural concepts and working capabilities, and provides public access to approved CTS materials, demonstrations, publications, and related resources.

## Repository Purpose

This repository exists to:

* Build and maintain the public CTS website
* Present an accessible explanation of CTS
* Publish approved architectural and governance projections
* Demonstrate selected CTS capabilities
* Link to authoritative repositories, publications, and project resources

This repository is a **publication surface**, not a governance authority.

## Authority Boundary

The website is a projection of approved CTS doctrine and implementation status.

It is not:

* The canonical CTS governance corpus
* A source of architectural authority
* An admission or governance mechanism
* A substitute for canonical policies, specifications, or decision records
* Evidence that displayed material has governance authority beyond its cited source

Where site content conflicts with the canonical CTS governance corpus, the governance corpus controls.

## What CTS Is

CTS is an architectural and operational system for preserving context, provenance, continuity, and meaning across sustained knowledge work.

Its constructive structure consists of:

* **Topology** — defines structures, relationships, boundaries, and flows
* **Tools** — provide execution surfaces and operational capabilities
* **Governance** — preserves authority, provenance, lifecycle, and coherence

These operate within CTS architectural constraints and contribute to the emergence of a **Cognitive Operating Environment (COE)**.

The website communicates this model. It does not define it.

## What Lives Here

This repository may contain:

* Website source files

* Deployment configuration

Canonical governance artifacts should not be authored or maintained here unless this repository is explicitly designated as their publication target.

## Content Rules

Published CTS claims should be:

* Traceable to an approved source
* Consistent with current CTS terminology
* Clearly distinguished from prototypes, demonstrations, and future direction
* Updated when controlling governance or architecture changes
* Removed or marked historical when superseded

The website may simplify presentation, but it must not silently alter architectural meaning.

## Repository Status

The site is under active development.

Content, presentation, links, demonstrations, and deployment mechanisms may change as CTS evolves. Public availability does not by itself establish architectural maturity, product readiness, or governance authority.

## Canonical Sources

Authoritative CTS doctrine, policies, decision records, and admitted governance artifacts reside in their designated canonical repositories.

This website should link to those sources where practical rather than duplicate them unnecessarily.
